# MediVU EMR 클릭형 데모 설계

## 목적

MediVU EMR은 투자·발표 청중이 3분 안에 Ambient Clinical Intelligence의 가치를 체험할 수 있는 정적 웹 데모다. 실제 환자나 의료 시스템을 연결하지 않고, 가상 내과 외래 진료에서 대화와 기존 기록이 키워드와 감별진단으로 정리되고 처방·오더 화면까지 이어지는 흐름을 보여준다.

## 사용자 경험

앱은 소개 페이지 없이 `가상 환자 A-001`의 진료 화면으로 시작한다. 전체 높이의 왼쪽 메뉴에서 `Ambient AI`가 첫 번째 활성 항목이며, 상단 환자 바에는 연령·성별·알레르기·과거력·방문 목적을 고정한다.

본문은 세 영역으로 나눈다.

1. 왼쪽은 의사와 환자의 대화를 순차적으로 표시하고 시작·일시정지·재시작을 제공한다.
2. 가운데는 출처별 키워드 보드와 확률순 진단 Top 5를 표시한다.
3. 오른쪽은 과거기록·검사·영상 근거를 원자료 형태로 유지한다.

키워드 보드는 위쪽의 AI·기존기록 근거와 아래쪽의 의사 직접 입력으로 구분한다. 키워드를 제거하면 원자료는 유지하되 진단 점수 계산에서만 제외한다. 의사가 임의로 입력한 키워드는 보존하며, 데모에 정의된 가중치가 없는 경우 진단 순위에는 영향을 주지 않는다.

진단 카드는 순위, 진단명, 후보별 독립 추정 확률, 근거 키워드, 진단코드, 병원 프로토콜, 의사 메모, 저장 오더세트 태그를 표시한다. 숫자는 합계 100%를 뜻하지 않으며 화면에 `AI 추정 확률 · 데모 데이터 · 최종 판단은 의료진`을 고정 표시한다.

진단을 선택하면 환자 문맥을 유지한 채 오더 화면으로 이동한다. 진단코드, 전달된 키워드, 시연용 처방·검사·추적관찰 항목은 미리 채워진다. 수정과 선택, 메모, 저장 성공 알림은 브라우저 메모리에서만 동작한다.

## 가상 증례

- 환자: 가상 환자 A-001, 58세 여성
- 방문 사유: 3일간 지속된 기침, 발열, 호흡곤란
- 과거력: 제2형 당뇨병, 경증 천식
- 활력·검사: 체온 38.2℃, 산소포화도 93%, 백혈구 및 CRP 상승
- 영상: 우하엽 침윤 소견
- 후보 진단: 지역사회획득폐렴, 급성 기관지염, 인플루엔자, COVID-19, 천식 악화
- 대표 선택: 지역사회획득폐렴(J18.9)

구체적인 약제 용량이나 임상적으로 유효한 처방 추천은 제공하지 않는다. 처방 항목은 `경구 항균제(시연용)`처럼 명확히 표시한다.

## 상태와 데이터 흐름

`DemoScenario`는 환자, 대화 이벤트, 초기 근거, 진단 후보와 오더 템플릿을 포함한다. 재생기가 대화 이벤트를 한 건씩 활성화하면 연결된 Ambient 키워드가 추가된다. 현재 활성 키워드 ID 집합을 `scoreDiagnoses`에 전달해 점수를 다시 계산하고 상위 다섯 후보를 정렬한다.

사용자 상태는 다음만 포함한다.

- 재생 위치와 재생 여부
- 활성·제외 키워드
- 의사가 추가한 키워드
- 진단별 사용자 태그
- 선택한 진단과 오더 편집값
- 저장 성공 알림

외부 API, 서버 저장소, 브라우저 영구 저장소는 사용하지 않는다. 전체 초기화는 모든 상태를 최초 가상 증례로 복원한다.

## 반응형과 접근성

1440×900과 1280×720에서 세 열을 유지한다. 1180px 미만에서는 오른쪽 근거 패널을 접이식으로 전환한다. 모든 상호작용은 키보드로 수행할 수 있어야 하며, 키워드 출처는 색상뿐 아니라 아이콘과 텍스트로도 전달한다. `prefers-reduced-motion`에서는 대화 및 순위 애니메이션을 줄인다.

## 안전 경계

- 모든 데이터는 가상이며 실제 환자정보를 포함하지 않는다.
- 실제 음성 전사, 생성형 AI, 의료기기 수준 확률, EMR 연동을 구현하지 않는다.
- 진단 확률과 처방·오더는 제품 사용 흐름을 설명하는 시연값이다.
- 최종 진단과 처방 책임이 의료진에게 있음을 지속적으로 표시한다.

## 완료 조건

처음 보는 발표자가 3분 안에 `대화 재생 → 키워드 변화 → Top 5 변화 → 진단 선택 → 오더 편집·저장`을 끊김 없이 시연할 수 있어야 한다. 단위·화면 테스트와 프로덕션 빌드가 통과하고, 기준 해상도에서 가로 넘침과 주요 접근성 문제가 없어야 한다.
