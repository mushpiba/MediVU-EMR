# MediVU EMR

MediVU EMR은 Ambient Clinical Intelligence가 진료 대화와 기존 임상 정보를 하나의 의사결정 화면으로 정리하는 과정을 보여주는 클릭형 데모입니다.

**이 프로젝트의 모든 환자·진단·확률·처방·오더 데이터는 가상 시연값입니다. 실제 의료기기, 진단 또는 처방 시스템이 아니며 실제 환자정보를 입력하면 안 됩니다.**

## 데모 보기

GitHub Pages 배포 후 다음 주소에서 열 수 있습니다.

<https://mushpiba.github.io/MediVU-EMR/>

## 핵심 흐름

- 가상 내과 외래 대화를 시작·일시정지·재시작합니다.
- 대화, 과거기록, 검사, 영상, 의사 입력 키워드를 출처별로 확인합니다.
- 키워드를 추가하거나 진단 근거에서 제외하면 Top 5 진단 후보와 독립 추정 확률이 갱신됩니다.
- 진단별 코드, 병원 프로토콜, 의사 메모, 저장 오더세트 태그를 편집합니다.
- 진단 후보를 선택해 미리 채워진 처방·검사·추적관찰 오더를 편집하고 브라우저 메모리에 저장합니다.

## 로컬 실행

Node.js 24와 npm을 권장합니다.

```powershell
npm.cmd install
npm.cmd run dev
```

표시된 로컬 주소 뒤에 `#/ambient`를 붙여 진료 화면을 엽니다.

### 검증

```powershell
npm.cmd run test:run
npm.cmd run build
npm.cmd run preview
```

## 3분 발표 순서

1. **0:00–0:30 — 환자 문맥**: 가상 환자 표시와 세 열 구조를 소개합니다. 오른쪽에서 과거력, 검사, 영상이 원자료로 유지되는 점을 강조합니다.
2. **0:30–1:20 — Ambient 재생**: `대화 재생`을 누릅니다. 문장이 표시될 때마다 대화 기반 키워드와 진단 확률이 바뀌는 장면을 보여줍니다.
3. **1:20–1:50 — 의사 통제**: 키워드 하나를 직접 추가하고, 기존 키워드의 삭제 버튼을 눌러 원자료는 남지만 진단 근거에서는 제외되는 동작을 보여줍니다.
4. **1:50–2:15 — 진단별 커스터마이즈**: 지역사회획득폐렴 카드에 짧은 의사 메모 태그를 추가합니다.
5. **2:15–2:50 — 오더 전환**: 지역사회획득폐렴을 선택해 자동 입력된 코드, 전달 근거, 저장 오더세트를 보여줍니다. 항목 하나를 제외하고 메모를 수정한 뒤 저장합니다.
6. **2:50–3:00 — 안전 경계**: 화면의 데모·의료진 최종 판단·서버 미전송 고지를 짚고 `데모 초기화`로 마칩니다.

## 구조

- `src/data`: 가상 환자, 대화, 진단 후보, 오더 fixture
- `src/domain`: 키워드 정규화와 진단 점수 계산
- `src/hooks`: 데모 재생 및 진료 상태
- `src/components`: Ambient 워크벤치와 오더 화면
- `docs/superpowers`: 승인된 설계와 구현 계획
- `docs/scenario-variants.md`: 응급실·호흡기내과 확장 방향

## 배포

`main` 브랜치에 푸시하면 `.github/workflows/deploy-pages.yml`이 테스트와 빌드를 수행한 뒤 `dist`를 GitHub Pages에 게시합니다. 저장소의 **Settings → Pages → Source**가 **GitHub Actions**로 설정돼 있어야 합니다.

## 범위 밖

- 실제 음성 전사 또는 생성형 AI

- 의료기기 수준의 확률 보정 및 임상적으로 유효한 처방 추천
